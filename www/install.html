You can then use the Air Carrier project forums to get in touch :)

Downloading Code

Getting the AirCarrier code from CVS is actually pretty easy, since there is an Eclipse project in the CVS repository. However you first need to get jME and jME Physics 2 CVS versions set up correctly. This shouldn't be too difficult following these instructions:

The first stage is to set up jME itself in eclipse, for this you can follow the excellent instructions on the jME wiki, please follow from the start, but you don't need to go through "Compiling Into JARs" or any later steps:

jME in Eclipse wiki page

When you have checked that you can run the jME HelloWorld test as described in the wiki, you are ready to install jme-physics as well. There is a wiki entry for this, but for some reason it doesn't quite match up (different project names, etc.). So just go back into the CVS repository view, find the "jmephysics" project under "HEAD" the same way you did for jme itself, right click and "Check out as...". The default settings are fine, to check out as a project in the workspace with name "jmephysics". Click "Finish".

When the update has finished, go back to the "Java" perspective in Eclipse, and find the jmephysics project. The project should be very nearly set up, since the project settings are in CVS, but you need to make one change. Right click the jmephysics project and select "Properties". Under "Java Compiler", uncheck the "Enable project specific settings" checkbox, click OK, and answer "Yes" if prompted for a rebuild.

Now you just need to check out aircarrier, this is very similar to checking out jme-physics.

Go back into the CVS repository view, find the "aircarrier" project under "HEAD" the same way you did for jme itself, right click and "Check out as...". The default settings are fine, to check out as a project in the worksapce with name aircarrier. Click "Finish".

When the update has finished, go back to the "Java" perspective in Eclipse, and find the aircarrier project. Expand "src", "net.java.dev.aircarrier", then right click the CarrierSimpleLevelManagers.java class, and select "Run as Java Application". You should see the AirCarrier test level. Use the mouse to steer, left and right mouse buttons to fire primary and secondary guns, and "WASD" to brake, boost and roll.

Common problems

If you have a small number of errors in one or more projects, it may well be that you are not up to date on all projects - to update them, select the project, right click, and select "Team -> Update". This will get any code changes from CVS. If there is still a problem, it is quite likely that aircarrier itself is not up to date with the most recent jme and jmephysics changes - in this case, please give the complete error you get, including line number etc., so that I can make sure aircarrier is updated. This should happen anyway, but I don't always remember to update from jme regularly :)

It's very easy to get the wrong compiler compliance settings for a project. This is why you need to uncheck the "Enable project specific settings" for the jme-physics project. If you have problems with many errors to do with classes not being generic, etc., you should check that all the projects have project specific settings disabled, and your workspace compiler settings are definitely set to 5.0 or 6.0, as described at the start of the jME in Eclipse wiki page

If applications run, but give an error about unsatisfied libraries, you need to make sure that you have followed the instructions for setting up the jars and native libraries on the jME in Eclipse wiki page
